# 均值回归算法设计文档

> 日期：2026-07-24
> 核心思路：不依赖市面封装指标（布林带/RSI/MACD 等），从价格结构本身定义"偏离"与"能量衰竭"。

---

## 一、数据输入

从 `tdx_quant.py` 的 `get_daily_kline_from_tdx(code, end_date)` 获取前复权日线：

| 字段 | 类型 | 说明 |
|------|------|------|
| `date` | datetime | 交易日 |
| `open` | float | 开盘价 |
| `high` | float | 最高价 |
| `low` | float | 最低价 |
| `close` | float | 收盘价 |
| `volume` | float | 成交量（手） |

**要求**：至少 120 个交易日数据，不足的跳过。

## 二、信号 A：滚动回归残差法

### 2.1 核心思想

股价不是围绕水平均值回归，而是围绕**它自身的短期趋势线**回归。因此先拟合趋势，再计算偏离。

### 2.2 算法步骤

```
对最近 T 个交易日做线性回归：

  价格序列:  P = [p_1, p_2, ..., p_T]
  时间索引:  t = [1, 2, ..., T]

  回归:  P_i = a * t_i + b + ε_i

  其中:
    a = 趋势斜率（正=上涨趋势，负=下跌趋势）
    b = 截距
    ε_i = 第 i 日的残差（偏差）
```

### 2.3 偏离度计算

```
残差序列:  ε = [ε_1, ε_2, ..., ε_T]
残差标准差: σ_ε = std(ε)

标准化残差（当日）:
  z_residual = ε_T / σ_ε
```

### 2.4 信号规则

| z_residual 范围 | 信号 | 强度 |
|----------------|------|------|
| ≤ -2.0 | 强向上回归（价格远低于趋势线） | 3 |
| -1.5 ~ -2.0 | 弱向上回归 | 1 |
| -1.0 ~ -1.5 | 关注 | 0 |
| -1.0 ~ +1.0 | 正常区间 | 0 |
| +1.0 ~ +1.5 | 关注 | 0 |
| +1.5 ~ +2.0 | 弱向下回归 | -1 |
| ≥ +2.0 | 强向下回归（价格远高于趋势线） | -3 |

### 2.5 参数

| 参数 | 默认值 | 说明 |
|------|--------|------|
| `reg_window` | 60 | 回归窗口（约 3 个月交易日） |
| `z_threshold_buy` | -1.5 | 买入偏离阈值 |
| `z_threshold_sell` | +1.5 | 卖出注意阈值 |
| `z_strong_buy` | -2.0 | 强买入阈值 |
| `z_strong_sell` | +2.0 | 强卖出阈值 |

### 2.6 优势

- 牛市不钝化：即使股价一直涨，偏离自身趋势线仍然会触发信号
- 数学干净：纯线性回归，无平滑/平滑参数争议
- 可解释：残差直接度量"价格相对自身近期趋势的位置"

## 三、信号 B：能量衰竭法

### 3.1 核心思想

连续同向大幅波动消耗了"势能"——当下跌力度逐步减弱且成交量萎缩时，表明卖盘衰竭，即将向上回归。

### 3.2 下跌能量定义

```
下行能量 = Σ( max(0, 跌幅_i) )  对于最近 N 日

其中 跌幅_i = (close_{i-1} - close_i) / close_{i-1}
```

只统计下跌日（close 下跌 > 0.5%），忽略小波动噪声。

### 3.3 能量衰竭判定

```
衰竭条件（全部满足）:
  1. 最近 M 日下行能量 > 阈值 E_min  （确认有显著下跌）
  2. 后半段（最近 M/2 日）日均下行能量 < 前半段 × 0.5  （下跌减速）
  3. 当日成交量 < 20日均量 × 0.8  （缩量确认）
```

### 3.4 评分规则

| 条件 | 得分 |
|------|------|
| 下行能量 > E_min | +1 |
| 下跌减速（后半段/前半段 < 0.5） | +1 |
| 缩量确认 | +1 |
| 最近一日跌幅 < 0.5%（跌不动了） | +1 |
| 收盘价接近当日最低（下影线短） | +1 |

总分范围 0~5，≥ 3 分视为有效衰竭信号。

### 3.5 参数

| 参数 | 默认值 | 说明 |
|------|--------|------|
| `energy_window` | 10 | 统计下行的窗口 |
| `drop_threshold` | 0.005 | 有效下跌阈值（0.5%） |
| `e_min` | 0.03 | 最小下行能量（约 3 个 1% 阴线） |
| `volume_ratio` | 0.8 | 成交量萎缩比例 |

## 四、信号融合逻辑

### 4.1 融合框架

两个信号各自输出**分数**和**信心等级**，按以下矩阵融合：

```
                 能量衰竭强 (≥4)    能量衰竭中 (3)    能量衰竭弱 (<3)
残差强回归 (z≤-2)     ★★★ 强烈买入     ★★☆ 买入        ★☆☆ 关注
残差弱回归 (-2<z≤-1.5)  ★★☆ 买入         ★★☆ 买入        ☆☆☆ 不操作
残差正常 (z>-1.5)     ☆☆☆ 不操作       ☆☆☆ 不操作       ☆☆☆ 不操作
```

### 4.2 最终信号输出

```python
{
  "code": "sz000001",
  "date": "2026-07-24",
  "signal": "buy" | "neutral" | "sell",
  "confidence": 1-5,           # 1=弱 2=关注 3=买入 4=强买入 5=强烈买入
  "z_residual": -2.3,          # 残差偏离度
  "energy_score": 4,           # 能量衰竭评分 0-5
  "details": {
    "a": 0.05,                 # 回归斜率
    "b": 10.2,                 # 回归截距
    "residual_std": 0.15,      # 残差标准差
    "recent_drop_energy": 0.08,# 下行能量
    "volume_ratio": 0.65,      # 当日量 / 20日均量
  }
}
```

## 五、验证方案

### 5.1 单股验证

选取 3-5 只典型股票，手动标注近期"超卖区域"，验证算法能否正确识别：

| 股票 | 特征 | 预期 |
|------|------|------|
| 深桑达A (000032) | RSI 24.9，连续下跌 | 应检出强回归信号 |
| 某趋势上涨股 | 正常上涨 | 不应检出信号 |
| 某横盘震荡股 | 正常波动 | 不应检出信号 |

### 5.2 批量扫描

对全市场或某个板块做一日扫描，输出信号分布统计：

- 总股票数 / 有信号数
- 信号强度分布（1-5分）
- 随机抽取10个信号做人工核验

### 5.3 回测（后续迭代）

初步验证通过后，可对历史数据跑回测，统计：

- 信号出现后 5/10/20 日收益率分布
- 胜率 / 盈亏比 / 最大回撤
- 不同参数组合下的表现
