# 板块 / 板块成分股 —— 替代数据源候选（排除 baostock/tushare/adata/finshare/akshare）

> 背景：当前项目 `juejing.py` 的板块数据走 **akshare**（`stock_board_industry_name_em` / `stock_board_industry_cons_em`），底层是**东方财富**；沙箱已确认屏蔽东方财富（push2his），概念板块（同花顺另一路）仍可用。
> 目标：在不使用 baostock / tushare / adata / finshare / akshare 的前提下，找到能拿「板块列表 + 板块成分股」且在沙箱可达的接口。

## 约束回顾

| 网络 | 沙箱状态 | 说明 |
| --- | --- | --- |
| 东方财富（push2his / datacenter） | ❌ 被挡 | akshare、efinance、adata 全系均失效 |
| 新浪财经 | ✅ 可达 | HTTP 公开接口 |
| 腾讯财经（gtimg） | ✅ 可达 | 实时行情为主 |
| 同花顺（i问财 / 概念） | ✅ 可达 | 记忆确认概念板块另一路可用 |

## 候选方案对比

| 方案 | 依赖 | 沙箱可达 | 板块列表 | 板块成分股 | 备注 |
| --- | --- | --- | --- | --- | --- |
| **① 新浪 `getHQNodeData`** | 仅 `requests` | ✅ | ✅ | ✅ | **首选，零依赖** |
| **② 同花顺 i问财 `pywencai`** | `pywencai`+Node.js | ✅ | ✅(自然语言) | ✅ | 需 cookie，调用重 |

## 首选方案 ①：新浪财经 `Market_Center.getHQNodeData`（零依赖、沙箱可达）

新浪行情中心提供**纯 HTTP GET** 接口，直接用 `requests` 即可，无需安装任何金融库。它同时返回**板块树**和**板块成分股**。

### 接口形态

```
基础地址:
http://vip.stock.finance.sina.com.cn/quotes_service/api/json_v2.php

取板块树（行业/概念/地域/指数 及其 node id）:
  /Market_Center.getHQNodes

取某板块的成分股:
  /Market_Center.getHQNodeData?page=1&num=100&sort=symbol&asc=1&node=<板块node>&symbol=&_s_r_a=init
```

### 关键参数（`getHQNodeData`）

| 参数 | 含义 | 说明 |
| --- | --- | --- |
| `page` | 页码 | 每页最多 100 条 |
| `num` | 每页条数 | 默认 40，最大 100 |
| `sort` | 排序字段 | 默认 `symbol`（股票代码） |
| `asc` | 排序方向 | `1` 升序 / `0` 降序 |
| `node` | 板块代码 | 见下方取值 |
| `symbol` | 指定股票 | 不填则返回该板块全部股票 |
| `_s_r_a` | 初始化参数 | 固定值 `init` |

### `node` 常用取值

| node | 含义 |
| --- | --- |
| `hs_a` | 沪深 A 股全部 |
| `all_hy` | 行业板块（一级/二级） |
| `all_gn` | 概念板块 |
| `all_dq` | 地域板块 |
| `zhishu_000001` | 上证指数成分股（指数类同理） |

> ⚠️ 确切的板块 node 名（如 `hy_xxxx` / `gn_xxxx`）以先调 `getHQNodes` 返回结果为准，避免硬编码记错。

### 可直接落地的代码

```python
import requests

SINA = "http://vip.stock.finance.sina.com.cn/quotes_service/api/json_v2.php"

def get_board_tree(node="all_hy"):
    """取板块树：行业/概念/地域/指数 及其 node id。"""
    url = f"{SINA}/Market_Center.getHQNodes"
    r = requests.get(url, params={"node": node}, timeout=10)
    return r.json()

def get_board_members(node, page=1, num=100):
    """取某板块的成分股。"""
    url = f"{SINA}/Market_Center.getHQNodeData"
    r = requests.get(url, params={
        "page": page, "num": num, "sort": "symbol", "asc": 1,
        "node": node, "symbol": "", "_s_r_a": "init",
    }, timeout=10)
    return r.json()

# 1) 先看行业板块有哪些（拿到每个板块的 node id）
tree = get_board_tree("all_hy")
for b in tree[:5]:
    print(b.get("node"), b.get("name") or b.get("symbol"))

# 2) 取某行业板块成分股（node 取自上一步）
members = get_board_members("hy_xxxx")   # 换成实际 node
for m in members:
    print(m.get("symbol"), m.get("name"), m.get("trade"))
```

返回为 JSON 数组，常见字段：`symbol`（如 `sz000001`）、`code`、`name`、`trade`（最新价）、`changepercent` 等。

## 方案 ②：同花顺 i问财 `pywencai`（自然语言、沙箱可达）

同花顺在沙箱可达。i问财用中文自然语言直接查板块/成分股，无需记接口参数，但**需要 Node.js + jsdom 环境**，免费版通常要 i问财 cookie。

```bash
pip install pywencai
npm install jsdom        # 在 Node.js 环境下
```

```python
import pywencai

# 直接问“某板块成分股”
res = pywencai.get(query="半导体板块成分股", query_type="stock", loop=True)

# 拿同花顺二级行业（指数代码 881 开头）
idx = pywencai.get(
    query="主力流入金额排序,指数代码881开头",
    query_type="zhishu", loop=True,
)
```

> 注意：免费接口常需登录 cookie（`cookie='...'`），且每次查询走 Node 执行 JS，比纯 HTTP 重。适合“偶尔查、要灵活”的场景。

## 方案 ③ / ④：TDX 协议直连（eltdx / pytdx）

- **eltdx**：概念/题材板块用 `client.helpers.stock_topics` + `client.helpers.topic_stocks`；行业/概念/风格/指数板块用社区 `tdx-mcp` 插件的 `tdx_get_block_info`。
- **pytdx**：`get_block_info()` / `get_and_parse_block_info()` 直接取板块成分。
- ❓ **关键未知**：两者都直连通达信 `7709` 主站，沙箱是否放行 `7709` 尚未验证。建议先 `eltdx-smoke` 或 ` telnet 116.205.183.150 7709` 探一下。

## 方案 ⑤：api51.cn（投资易，干净 REST，可达性待验）

```
所有板块:     http(s)://stock.api51.cn/block/info?finance_mic=XBHS        # XBHS=沪深板块
板块合集排序: http(s)://stock.api51.cn/sort?en_hq_type_code=XBHS.GN     # GN=概念 HY=行业 DY=地域
股票所属板块: http(s)://stock.api51.cn/block/query?prod_code=000001.SZ
板块内股票:   http(s)://stock.api51.cn/sort?en_prod_code=610300.XBHS
```

字段集对照：`XBHS.GN` 沪深概念、`XBHS.HY` 沪深行业、`XBHS.DY` 沪深地域、`XBHS.ZJHHY` 证监会行业。
❓ 该域名在沙箱是否可达未验证，需先 curl 探一下。

## 不建议

- **东方财富 datacenter-web 接口**：与 akshare/adata 同源（东方财富），沙箱已挡，且你已排除 adata。
- **efinance**：虽未显式排除，但底层东方财富 → 沙箱失效；且其 `stock.get_realtime_quotes` 不提供板块成分。
- **xtquant(QMT) / 掘金 gm**：需券商/量化终端常驻，非免费公开接口（你 `juejing.py` 已用 gm 取 K 线，但板块仍需终端）。

## 结论与建议

1. **直接替换 akshare 板块数据 → 用方案 ① 新浪 `getHQNodeData`**：零依赖、沙箱可达、同时覆盖板块列表与成分股，是最小改动路径。
2. 若你要的是**同花顺概念/题材口径**且能接受 Node+cookie → 方案 ② `pywencai`。
3. 若要走 **TDX 协议拿行业/概念/风格/指数完整板块树** → 方案 ③ eltdx+tdx-mcp，但**先验证 7709 在沙箱可达**。
4. api51.cn 作为干净 REST 备选，可达性待验。

> 下一步可做的验证：在沙箱里 `curl` 探 `vip.stock.finance.sina.com.cn` 与 `116.205.183.150:7709`，确认 ① 和 ③ 的实际可达性，再决定最终切换方案。
